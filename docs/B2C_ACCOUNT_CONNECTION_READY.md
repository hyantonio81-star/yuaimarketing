# B2C 계정 연동 준비 — 연동 완료 후 작업 가능 설정 기획

## 목표

**계정(이커머스 채널) 연동 설정만 마치면 B2C Pillar의 모든 기능을 바로 사용할 수 있도록** 준비된 상태를 유지합니다.

## 선행 조건

| 단계 | 내용 | 완료 시 |
|------|------|--------|
| 1 | **이커머스 채널 연동** (Shopify 등) | 재고 동기화·최적가·프로모션 등에서 실채널 데이터 사용 가능 |
| 2 | (선택) 조직·국가 스코프 설정 | 다조직·다국가 재고/연동 구분 |

연동 전에는 **시뮬레이션/스텁 모드**로 동작하며, UI에 "연동 후 사용 가능" 안내가 표시됩니다.

## 연동 경로 (사용자 플로우)

1. **B2C Commerce** (`/b2c`) 접속
2. 상단 **「이커머스」** 버튼 클릭 → `/b2c/ecommerce` 이동
3. 채널 선택 후 **연동** (Store URL, API 키 등 설정)
4. 연동 완료 시 B2C 메인으로 복귀 → 모든 섹션에서 해당 채널 사용 가능

또는 **설정 > 연동**에서 이커머스 연동을 관리할 수 있도록 라우트가 있을 경우 동일 플로우 적용.

## 연동 완료 후 사용 가능한 기능

| 기능 | 설명 | 연동 시 동작 |
|------|------|--------------|
| 재고 동기화 | SKU·수량 변경 반영 | 연동된 채널(Shopify 등)에 반영 요청 |
| 주문 처리 | 주문 상태·처리 | 실주문 데이터 기준 |
| 최적가 | 제품별 가격 추천 | 연동 채널·경쟁사 데이터 반영 |
| 프로모션 플랜 | 채널별 프로모션 시나리오 | 선택 채널에 맞춘 시나리오 |
| 리뷰 분석 | 상품 리뷰·감정 분석 | 연동 채널 리뷰 수집 후 분석 |
| 시뮬레이션 상품 | 최적가/프로모션/리뷰용 상품 선택 | 실 SKU/상품 목록 사용 가능 |
| 부정 리뷰 대응 | 부정 리뷰 탐지·대응안 | 실 리뷰 데이터 |
| 이탈 방지 | 이탈 위험 고객·캠페인 | 실 구매·행동 데이터 |
| 추천 | 상품·채널 추천 | 실 데이터 기반 |

## 개발 측 준비 상태 (체크리스트)

- [x] B2C 페이지에서 **연동 여부** 표시 (`hasConnection`, 연동 채널 뱃지)
- [x] **미연동 시** 시뮬레이션 모드 안내 배너 노출
- [x] **연동 후 사용 가능** 기능이 연동 채널(`connectedChannels`)을 받아 동작하도록 구성
- [x] 이커머스 연동 API (`GET/POST/DELETE /b2c/connections`) 및 프론트 `ecommerceApi` 연동
- [ ] (선택) 설정 전용 라우트 `/settings/connections`에서 B2C 연동 진입점 제공
- [ ] (선택) 연동 완료 시 온보딩 툴팁 또는 단계별 가이드

## 프론트 표시 규칙

- **연동됨**: 상단에 "Shopify(또는 채널명) 연동됨" 뱃지 + 각 섹션에서 해당 채널 사용 가능 안내
- **미연동**: "계정 연동을 완료하면 아래 모든 기능을 사용할 수 있습니다" + **이커머스** 링크로 연동 유도

## 백엔드 의존

- `GET /b2c/connections` — 연동 목록 (조직별)
- `POST /b2c/connections` — 채널 연동
- `DELETE /b2c/connections?channel=...` — 연동 해제
- Supabase 사용 시 `b2c_channel_connections` 테이블; 미사용 시 메모리 fallback (재시작 시 초기화)
