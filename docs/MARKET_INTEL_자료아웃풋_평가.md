# 시장인텔 자료 아웃풋 평가

## 1. 현재 구조 요약

시장인텔 화면에서 **“자료가 나가는”** 구간은 크게 두 가지입니다.

| 구간 | 화면 위치 | 현재 아웃풋 형태 | 파일 다운로드 |
|------|-----------|------------------|---------------|
| **시장 리포트** | 시장 리포트 출력 설정 → 리포트 생성 | 설정 저장 + 생성 API 호출 | ❌ 없음 |
| **세분화 분석 결과** | 세분화 분석의뢰 → 결과 보기 | 테이블·카드 (화면 표시만) | ❌ 없음 |
| **시장 뉴스 요약** | 시장 뉴스 요약 | 기사 목록 + 원문 링크 | ❌ 없음 (링크만) |

---

## 2. 시장 리포트 아웃풋 (핵심)

### 2.1 구현된 것

- **출력 형식 선택**: PDF, Word(DOCX), Excel, JSON
- **출력 설정 저장**: 형식·스코프·스케줄·포함 섹션·언어·등급·페이지 수 → `PUT /report-settings`로 저장
- **리포트 생성 버튼**: `POST /generate-report` 호출
- **API 응답**: `job_id`, `status: "completed"`, `message`, `format` (선택한 형식) 반환

### 2.2 아직 없는 것 (자료 아웃풋 관점)

| 항목 | 설명 |
|------|------|
| **실제 파일 생성** | PDF/Word/Excel/JSON 파일을 만드는 파이프라인이 연결되어 있지 않음. |
| **다운로드 URL** | 응답의 `download_url`은 항상 `null`. |
| **비동기 작업** | 생성이 오래 걸릴 경우를 위한 `pending → completed` 상태·폴링/웹소켓 없음. |

즉, **“설정은 저장되고, 생성 버튼은 동작하지만, 다운로드할 수 있는 자료(파일)는 아직 나오지 않는 상태**입니다.

### 2.3 백엔드 코드 기준

```ts
// marketIntelService.ts - generateMarketReport()
return {
  job_id,
  status: "completed",
  message: "Report generation pipeline is not yet connected. Your options have been saved; ...",
  download_url: null,   // ← 실제 파일 링크 없음
  format: options.format,
};
```

문서에도 `"리포트 생성 파이프라인 연동 예정"`으로 되어 있습니다.

## 3. 그 외 화면별 아웃풋

### 3.1 세분화 분석 결과

- **시장 장악 포인트**, **관련 업체 리스트**, **사용 데이터 소스**를 화면에만 표시.
- 테이블/카드 내용을 **엑셀·CSV 등으로 내보내기** 기능은 없음.

### 3.2 시장 뉴스 요약

- 기사 제목·요약·출처·날짜 표시.
- 링크 있는 기사는 **클릭 시 원문으로 이동** (이미 구현됨).
- **뉴스 목록을 PDF/엑셀 등으로 한 번에 받는** 기능은 없음.

### 3.3 툴·서비스·결과물·데이터 소스

- 정적 설명 + 소스 링크 수준.
- “자료 아웃풋”이라기보다는 **설정/참고 정보**에 가깝습니다.

---

## 4. 종합 평가

| 항목 | 평가 | 비고 |
|------|------|------|
| **출력 형식 옵션** | ✅ 양호 | PDF/Word/Excel/JSON 선택 가능, 설정 저장됨. |
| **리포트 실제 생성** | ❌ 미구현 | 파이프라인 미연동, 파일 생성·다운로드 없음. |
| **다운로드 경험** | ❌ 없음 | `download_url` 미제공, “자료 받기” 불가. |
| **세분화·뉴스 등** | △ 부분적 | 화면 조회·링크 이동만 가능, 파일 아웃풋 없음. |

**한 줄 요약**:  
시장인텔의 **자료 아웃풋**은 “어떤 형식으로 낼지 설정하고, 생성 요청까지 보내는” 단계까지는 갖춰져 있으나, **실제 리포트 파일 생성 및 다운로드는 아직 없고**, 세분화/뉴스도 “화면 보기·링크” 수준입니다.

---

## 5. 개선 제안 (자료 아웃풋 강화)

1. **시장 리포트**
   - 리포트 본문을 만드는 **실제 생성 파이프라인** 구현 (선택한 섹션·언어·등급·페이지 수 반영).
   - 생성된 파일을 저장(로컬/스토리지)하고, 응답에 **`download_url`** (또는 파일 스트리밍) 제공.
   - 생성 시간이 길어질 수 있으면 **비동기 작업(job_id 폴링)** 으로 상태 조회 후 완료 시 다운로드 링크 노출.

2. **세분화 분석 결과**
   - “결과 보기” 영역에 **엑셀/CSV 내보내기** 버튼 추가 → 테이블·시장 장악 데이터를 파일로 받을 수 있게 함.

3. **시장 뉴스 요약**
   - (선택) “현재 보이는 뉴스 목록을 PDF/엑셀로 다운로드” 기능 추가.

4. **프론트**
   - `generate-report` 응답에 `download_url`이 있으면 **다운로드 링크/버튼**을 노출해, “자료가 여기서 나온다”는 느낌을 명확히 함.

이렇게 단계적으로 연결하면 시장인텔의 **자료 아웃풋**이 “설정 + 생성 요청”을 넘어 **실제 다운로드 가능한 자료**까지 제공하는 형태로 정리됩니다.
