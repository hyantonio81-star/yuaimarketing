# 인기 상품 해석 + 멀티 포맷·멀티 채널 홍보 기획

인기 상품을 **해석·선별**한 뒤, 한 제품을 **영상·블로거·SNS 등 여러 홍보 매개체**로 만들고 **여러 분야·채널**로 배포하는 흐름을 정리한 기획서입니다.

**관점**: **제휴(affiliate) 상품** 기준 — 선정·제작·배포는 모두 “제휴 상품 1건” 단위로 동작하며, 모든 상품 링크에는 제휴 추적 ID가 부착된다 ([AFFILIATE_TRACKING_LINKS.md](./AFFILIATE_TRACKING_LINKS.md)).

---

## 1. 인기 상품에 대한 해석

| 관점 | 설명 |
|------|------|
| **시장 인기** | 이미 잘 팔리거나, 검색·트렌드에 오른 제품 (Bestsellers, Movers & Shakers, 카테고리 인기 등) |
| **AI 팀이 높게 평가한 제품** | 마진·수익성, 경쟁 강도, 키워드 적합도, 규제/컴플라이언스, 시장 타이밍 등을 점수화해 **자체 점수가 높은** 제품 |

**정의**: “인기 상품” = **시장 인기** + **AI 팀의 높은 평가**를 합쳐, **홍보할 가치가 있다고 판단한 제품**만 다음 단계(제작·배포)로 넘긴다.

---

## 2. 현실적인 플로우

한 번 선정된 “인기·고평가 제품”은 **한 제품 → 여러 포맷 → 여러 채널·분야**로 가는 구조가 맞다.

```
[1. 선정]
  시장 인기 데이터 + AI 팀 평가(점수·마진·트렌드·컴플라이언스)
       → “홍보할 제품” shortlist

[2. 제작]
  한 제품당
  - 영상 (Shorts / TikTok / Reels용)
  - 블로거용 글 + 이미지
  - SNS용 카피 (Threads, 인스타 등)
  - (선택) 랜딩 전용 요약·CTA

[3. 배포]
  - 블로거/블로그
  - 영상: 유튜브, 틱톡, 인스타 등
  - SNS: Threads, 인스타
  - “여러 분야”: 타깃(니치·국가·언어)별로 톤·채널만 바꿔 재활용
```

---

## 3. 시스템에 넣을 수 있는 구체 요소

### 3.1 선정 단계

| 현재 | 보강 |
|------|------|
| Bestsellers / Movers & Shakers, 쉬인/테무 등 수집만 있음 | **AI 팀 평가 점수** (마진, 경쟁강도, 키워드 적합도, 규제 리스크 등)를 한 번에 계산 |
| — | “인기 + 고평가”만 필터링하는 **홍보 후보 리스트** API/화면 |

- **입력**: 마켓플레이스·카테고리·소스(베스트셀러/무버스 등).
- **출력**: 제품 목록 + 각 제품별 **홍보 가치 점수**(popularity_score, ai_team_score, composite_score). 이 점수로 shortlist 생성.

### 3.2 제작 단계

- **한 제품 ID 기준**으로 다음을 묶어서 생성:
  - **영상**: Shorts 파이프라인(스크립트·썸네일·TTS·영상).
  - **블로거**: 콘텐츠 자동화(블로그 본문 + “Buy here” 링크).
  - **SNS**: Threads Commerce / 콘텐츠 자동화 카피 + 이미지.
- **Job/결과 모델**: 같은 제품에 대해 “영상 job + 블로그 job + SNS job”이 **product_id(또는 asin 등)** 로 연결되도록 설계.

### 3.3 배포 단계

| 매개체 | 채널 | 비고 |
|--------|------|------|
| 블로거 | 블로그 | 콘텐츠 자동화 → Blogger |
| 영상 | 유튜브 | Shorts 파이프라인 (현재 구현됨) |
| 영상 | 틱톡·인스타 | 추후 연동 (필드·대비 설계) |
| SNS | Threads, 인스타 | Threads Commerce·콘텐츠 자동화 |
| “여러 분야” | 타깃별 채널 | 국가·언어·니치별 톤 조정, 같은 콘텐츠를 다른 채널/계정으로 재사용 |

## 6. 적용 완료 사항 (시스템 반영)

| 항목 | 적용 내용 |
|------|-----------|
| **선정 단계** | `affiliateProductScoreService.ts`: 제휴 상품에 `popularity_score`, `ai_team_score`, `composite_score` 부여 (휴리스틱). `getPromoShortlist()`로 점수 정렬·min_score 필터. |
| **Shortlist API** | `GET /api/seo/threads-commerce/shortlist?marketplace=&limit=&min_score=` — 홍보 후보 목록 반환. |
| **product_id 연동** | Threads Commerce `PipelineResult`, Content Automation `ContentAutomationResult`에 `product_id` 필드 추가. 같은 제품의 영상·블로그·SNS job을 product_id로 연결 가능. |

추가 개선: 프론트 shortlist 화면, “이 제품으로 블로그·SNS·영상 생성” 트리거, 제품 기반 Shorts 입력 경로는 추후 단계로 확장.

## 7. 구현 우선순위 제안

| 순서 | 내용 | 결과 |
|------|------|------|
| 1 | **홍보 후보 점수** — 수집된 상품에 AI 팀 평가 점수 부여, shortlist API | ✅ 적용 (`/shortlist`, `affiliateProductScoreService`) |
| 2 | **1제품 → 멀티 포맷** — job 결과에 product_id 공통 부여 | ✅ 적용 (PipelineResult, ContentAutomationResult에 product_id) |
| 3 | **배포 채널 확장** — 영상 유튜브 유지, 틱톡·인스타 대비; SNS 기존 연동 | 추후 |

이 문서를 기준으로 “인기 상품 해석 → 멀티 포맷 제작 → 멀티 채널·여러 분야 홍보” 플로우를 단계적으로 적용하면 됩니다.
