# /api/* 404 해결 가이드

## 증상
- https://yuaimarketing.vercel.app/ → **정상** (메인 페이지)
- https://yuaimarketing.vercel.app/api/health → **404**

## 원인
Vercel이 빌드할 때 **프로젝트 루트**에 `api/` 폴더가 없음.  
저장소 구조가 `Music/Mark01/nexus-ai/api/` 이므로, **Root Directory**를 반드시 `Music/Mark01/nexus-ai`로 설정해야 합니다.

---

## 해결 (둘 중 하나)

### 1) 대시보드에서 수동 설정
1. https://vercel.com → **yuaimarketing** 프로젝트
2. **Settings** → **General**
3. **Root Directory** → **Edit** → **`Music/Mark01/nexus-ai`** 입력 → **Save**
4. **Deployments** → **Redeploy** (최신 배포 또는 Production)

### 2) 토큰으로 스크립트 실행
```powershell
cd Music\Mark01\nexus-ai
$env:VERCEL_TOKEN="여기에_토큰"
npm run vercel:set-root
```
실행 후 Vercel 대시보드에서 **Redeploy** 한 번 실행.

---

## 확인
설정 후 재배포가 끝나면:
- https://yuaimarketing.vercel.app/api/health → **200** + `{"status":"ok",...}`

---

## Production 브랜치
- **Settings** → **Git** → **Production Branch**가 **master**인지 확인.
- master에 `Music/Mark01/nexus-ai` 구조가 있으므로, Root Directory와 함께 맞추면 API 404가 사라집니다.
